canvas의 렌더링 프로세스 

-> 캔버스의 call function
rendering process

메인스레드에서 call skia function (that makes "record" data) 
(즉 이 개념을 이해하려면 record라는 개념을 이해해야 한다)

-> webgl or webgpu based canvas completely bypass expensive main thread DOM pipeline and go straight to GPU rendering

-> skia graphite(skia gpu backend) 이게 어디에서 실행되나 ?
-> dawn (webgpu implementation) 
-> vulkan, metal or d3d12 (os level abstraction layer) 
-> gpu driver api call (eg. nvidia rtx driver api call) 
-> render to monitor

---

Introducing Skia Graphite
https://blog.chromium.org/2025/07/introducing-skia-graphite-chromes.html
chromium rendering engine skia now use new backend called "Graphite"
Graphite use dawn ( webgpu implementation) for rasterization

---

in the future, skia will record each tile per thread
then, recorded information will pass to dawn (webgpu implementation) simultaneously
